# Artifact Generation Architecture

## Executive Summary

The artifact generation system establishes a standardized workflow for persisting persona outputs as version-controlled markdown files, organized by persona with consistent templates and naming conventions.

## Main Content

The artifact generation system transforms valuable persona outputs into persistent,
version-controlled markdown files. This system is designed for:

- **Research Reports**: Deep-dive analyses with cited sources
- **Technical Plans**: Architecture decisions and implementation paths
- **Operational Runbooks**: Standard operating procedures and recovery guides
- **Market Analysis**: Competitive intelligence and investment research

## Design Decisions

1. **Local-first storage**: Artifacts live in `artifacts/<persona>/` within the
   platform repository, ensuring they are version-controlled alongside the
   platform itself.
2. **Self-contained files**: Each artifact is a complete markdown document that
   can be read, searched, and shared independently.
3. **Persona-namespaced directories**: Each persona agent writes only to its own
   directory, preventing cross-contamination and maintaining clean ownership.
4. **Script-driven generation**: The `scripts/generate-artifact.sh` script
   enforces the markdown template, naming convention, and storage rules.

## Workflow

```
User Request
  → Persona Agent executes research/analysis
  → Agent generates markdown content
  → Agent calls generate-artifact.sh with persona, title, content
  → Script validates persona, formats template, writes file
  → Script returns artifact path
  → Agent confirms save to user with title, path, and summary
```
