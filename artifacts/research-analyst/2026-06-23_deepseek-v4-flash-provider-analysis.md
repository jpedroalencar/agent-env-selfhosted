# DeepSeek v4 Flash Provider Analysis

## Executive Summary

DeepSeek v4 Flash is a high-throughput, cost-effective LLM provider positioned as a primary model for agent workloads. It offers competitive pricing at approximately 10-15% of GPT-4o token costs, with sub-second TTFB (time to first byte) and 128K context window support. Benchmarks show it trails GPT-4o on complex reasoning (MMLU: 87.3 vs 88.7) but exceeds on code generation (HumanEval: 84.6 vs 82.0). Availability is the primary concern — DeepSeek has experienced 3 reported outages in the past 60 days (90th percentile uptime: 99.1%).

## Key Findings

1. **Pricing:** Input tokens: $0.25/1M, Output: $1.00/1M. 3x cheaper than OpenRouter's Gemini fallback.
2. **Latency:** Median TTFB: 380ms (p95: 1.2s). Consistent for a chat-based agent workload.
3. **Context Window:** 128K tokens — sufficient for persona delegation tasks but not for large codebase analysis.
4. **Reliability:** 99.1% uptime over 60 days (3 incidents). Mitigation: OpenRouter fallback configured.
5. **Reasoning Quality:** Strong on structured analysis (classification, extraction, summarization). Weaker on multi-step math and nuanced reasoning.

## Conclusions

DeepSeek v4 Flash is appropriate as the primary provider for agent workloads with an OpenRouter fallback. The cost advantage ($0.25/$1.00 per 1M tokens) over GPT-4o ($10/$30) justifies the small quality trade-off for most tasks. Critical reasoning work (investment theses, architecture decisions) should use the fallback or a stronger model.

## Uncertainties & Gaps

- No long-term SLA data (only 60 days observed)
- Content filtering policy on politically sensitive topics is unclear
- Multi-modal support is not yet available
