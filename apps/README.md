# Applications Directory

## Purpose

The `apps/` directory hosts application definitions, configurations, and metadata for applications built *on top of* the Agent Platform. It is the boundary between **infrastructure** (the platform itself) and **products** (applications built using the platform).

## Separation of Concerns

```
agent-env-selfhosted/          ← Platform repository
├── agent/                     ← Agent configuration (personas, skills, memory schemas)
├── artifacts/                 ← Curated outputs generated by personas
├── docs/                      ← Platform documentation
├── infra/                     ← Infrastructure configuration
├── scripts/                   ← Automation and utility scripts
└── apps/                      ← ← Application definitions live here
    ├── README.md              ← This file
    └── <app-name>/            ← Individual application directories
```

## Application Repository Strategy

Applications that require their own development lifecycle, CI/CD, or independent deployment should live in **separate repositories**. The `apps/` directory here serves as:

- A **registry** of known applications built on the platform
- A place for **thin wrappers** that reference external repositories
- A home for **application-specific configuration** that the platform needs to know about (e.g., which persona to invoke, where to deliver artifacts)

### When to keep an application inside this repository

| Criteria | Keep here |
|----------|-----------|
| Small tool/script (< 500 LOC) | ✅ |
| Tightly coupled to platform config | ✅ |
| Single-file utility | ✅ |
| Prototype / exploration | ✅ |
| No independent deploy needed | ✅ |

### When to create a separate repository

| Criteria | Separate repo |
|----------|---------------|
| Large codebase (5000+ LOC) | ✅ |
| Needs independent CI/CD | ✅ |
| Shared across multiple platforms | ✅ |
| Public-facing product | ✅ |
| Has its own user base | ✅ |
| Requires independent versioning | ✅ |

## Recommended Structure for an Application

```
apps/<app-name>/
├── README.md              # What it does, how to use it
├── config.yaml            # Platform-specific configuration
├── requirements.txt       # Python/system dependencies
├── src/                   # Source code
├── tests/                 # Tests
└── scripts/               # Build/deploy scripts
```
