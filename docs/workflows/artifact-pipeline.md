# Artifact Pipeline

> **Audience:** Persona agents executing the artifact generation workflow.
> **Last updated:** 2026-06-23

## Overview

The artifact pipeline transforms valuable persona outputs into persistent, version-controlled markdown files. It is the mechanism by which ephemeral chat responses become durable platform assets.

```
User Request → Persona Execution → Evaluate Output → Generate Artifact → Register → Report
```

## When to Generate an Artifact

### Generate ✅

| Type | Examples | Persona |
|------|----------|---------|
| Research reports | Deep-dive analysis, competitive intelligence, literature reviews | Research Analyst |
| Market analysis | Sector overviews, trend analysis, regulatory landscapes | Research Analyst |
| Valuation reports | Company analysis with P/E, PEG, bull/bear cases | Financial Analyst |
| Earnings reviews | Quarterly results, key metrics vs. expectations | Financial Analyst |
| Portfolio analysis | Allocation reviews, rebalancing suggestions, risk assessments | Financial Analyst |
| Architecture documents | Design decisions, trade-off analysis, system diagrams | Dev |
| Implementation plans | Scope, steps, dependencies, timeline | Dev |
| Debugging reports | Root cause analysis, resolution, lessons learned | Dev |
| Technical runbooks | Deployment procedures, recovery guides, SOPs | Dev / Operations Manager |
| Operational procedures | Maintenance schedules, escalation paths, handoff protocols | Operations Manager |
| Roadmaps | Phases, milestones, priorities, dependencies | Operations Manager |
| Status reports | Cross-persona progress summaries, blocker tracking | Operations Manager |

### Skip ❌

- Short answers (under ~500 words, no substantive analysis)
- Single commands or configuration snippets
- Clarifications or follow-up questions
- Status messages ("working on it", "completed")
- Casual conversation
- Error messages without root cause analysis

## Artifact Structure

Every artifact follows this template:

```markdown
# Title

## Executive Summary

One to three paragraphs summarizing the key findings, conclusions, or recommendations.

## Main Content

The substantive body of the artifact. May include:

- Analysis and findings
- Data and evidence
- Code blocks or configuration snippets
- Tables and comparisons
- Diagrams (Mermaid, ASCII)
- Lists and procedures

## Sources

When applicable, list cited sources:

- [Title](URL) — Publisher, Date
- [Title](URL) — Publisher, Date

## Generated By

- **Persona:** Persona Name
- **Timestamp:** YYYY-MM-DD HH:MM:SS UTC
```

## File Naming Convention

```
artifacts/<persona>/YYYY-MM-DD_short-kebab-case-title.md
```

Examples:

| Title | Filename |
|-------|----------|
| DeepSeek Provider Analysis | `artifacts/research-analyst/2026-06-23_deepseek-provider-analysis.md` |
| AAPL Valuation Report | `artifacts/financial-analyst/2026-06-23_aapl-valuation-report.md` |
| LXD Backup Architecture | `artifacts/dev/2026-06-23_lxd-backup-architecture.md` |
| Phase 2 Roadmap | `artifacts/operations-manager/2026-06-23_phase-2-roadmap.md` |

## Registry

The artifact registry at `artifacts/index.md` is an append-only table tracking every generated artifact:

```
| Date | Title | Persona | Path | Summary |
```

Each artifact generation appends a single row. The registry enables:

- Quick discovery of all generated artifacts
- Cross-referencing between related artifacts
- Search and retrieval by persona, date, or topic

## Procedure (for Persona Agents)

### Step 1: Evaluate

After producing your output, check whether it meets the artifact criteria for your persona. Ask:

- Is this output substantive? (> ~500 words or significant analytical/technical content)
- Would this output be valuable to reference in the future?
- Does it fall into one of the "Generate" categories above?

If yes → proceed. If no → skip artifact generation and deliver the response normally.

### Step 2: Generate the artifact

1. Derive a short kebab-case title from the output's topic
2. Construct the filename: `artifacts/<persona>/YYYY-MM-DD_kebab-title.md`
3. Write the artifact using the template above
4. Include the executive summary, all main content, sources, and generated-by metadata

### Step 3: Register in the index

Append a row to `artifacts/index.md`:

```
| 2026-06-23 | Artifact Title | Persona Name | artifacts/persona/filename.md | One-sentence summary |
```

### Step 4: Report to the user

Return:

- The artifact title
- The artifact path
- A short executive summary (one to three sentences)

Do NOT send the full artifact text through Telegram if it has already been saved — the user can read it at the file path.

## Retrieval

To retrieve an artifact:

1. **By registry lookup:** Read `artifacts/index.md` and find the relevant row. The path column links directly to the file.
2. **By directory browse:** List files under `artifacts/<persona>/` sorted by date (filenames are date-prefixed).
3. **By search:** Use grep/search tools to find content across all artifact files.

## Integration Points

- **Persona definitions:** Each persona file (`agent/personas/*.md`) includes an Artifact Generation section with persona-specific criteria and the procedure.
- **Repository structure:** The `artifacts/` directory is tracked in Git. Artifacts are version-controlled alongside the platform.
- **Telegram delivery:** Artifact notifications are lightweight — title, path, summary — not the full file content.

## Persona-Specific Overrides

### Research Analyst

Generates artifacts for: structured research briefs, competitive comparisons, product evaluations, regulatory summaries, literature reviews.

### Financial Analyst

Generates artifacts for: company valuations, earnings reviews, portfolio analyses, risk assessments, sector overviews.

### Dev

Generates artifacts for: architecture decisions, implementation plans, debugging reports, technical runbooks, configuration documentation.

### Operations Manager

Generates artifacts for: operational runbooks, roadmaps, status reports, procedural definitions, coordination plans.
